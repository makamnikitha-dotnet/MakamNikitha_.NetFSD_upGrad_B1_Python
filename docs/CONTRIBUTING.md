# Contributing Guidelines

Thank you for your interest in contributing to the **Multi-Page E-Learning Platform** project.

## Code Standards
- **HTML**: Use semantic tags (e.g., `<header>`, `<main>`, `<footer>`) for better accessibility.
- **CSS**: Prefer Bootstrap utility classes for layout; use `style.css` for custom components.
- **JavaScript**: Write modular code and use ES6+ features where applicable.
- **Comments**: Include brief comments for complex logic in `app.js` and `storage.js`.

## How to Contribute
1. **Fork the repository**: Create your own copy of the project on GitHub.
2. **Create a branch**: Use descriptive names like `feature/new-quiz` or `fix/navigation-bug`.
3. **Commit your changes**: Provide clear, concise commit messages.
4. **Push and Pull Request**: Push your branch and open a PR for review.

## Reporting Issues
If you encounter a bug or have a suggestion, please open an **Issue** on the repository with:
- A clear title and description.
- Steps to reproduce (if it's a bug).
- Screenshots (if applicable).
